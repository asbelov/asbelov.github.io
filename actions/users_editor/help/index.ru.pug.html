Users editor

Users editor позволяет создать нового или отредактировать существующие данные пользователей системы
Возвращаемое значение: нет

Описание интерфейса

Сверху находятся элементы для ввода или изменения данных о пользователе.

  1. User name - имя пользователя, которое будет вводиться для входа в систему
  2. Full user name or user description - Полное имя пользователя или описание, если это технический пользователь
  3. User password и Confirm password - поля для ввода и подтверждения пароля
  4. User roles - выбор ролей пользователя для получения необходимых прав в системе

Далее расположены настройки средств связи с пользователем. Для каждого средства связи используется два поля:

  1. Адрес по которому можно связаться с пользователем, используя данное средство связи. Например, это может быть email, номер телефона и так далее.
  2. Приоритет устанавливает в каких случаях стоит использовать данные средство связи. Например, можно настроить, чтобы при событиях с низкой важностью в качестве средства связи использовалась электронная почта, а при событиях с высокой важностью пользователю отправлялись СМС.

Ниже находится список пользователей. Первая строка "NEW USER" используется для заведения нового пользователя. При выборе остальных строк в поля загружается информация пользователе, которую можно отредактировать. Удалить пользователя можно с помощью "x". Системного пользователя (обычно с именем SYSTEM) удалить нельзя. От его имени производится автоматический запуск задач.

Список ролей пользователей, установленный по умолчанию

  1. Administrators - роль с полными правами
  2. Business - роль с правами на просмотр объектов и создание задач, которые в дальнейшем может выполнить Administrator
  3. Watcher - роль с правами на просмотр объектов
  4. Guest - отсутствуют какие либо права. Пользователь без авторизации получает права этой роли.

Список пользователей, установленный по умолчанию

По умолчанию в системе присутствуют пользователи admin (роль Administrators), business (роль Business), guest (роль Guest), system (роль Administrators) и watcher (роль Watcher). Пароли для admin, business и watcher установлены такие же, как и имена пользователей. После первого запуска системы администратору необходимо поменять пароли для пользователей, либо удалить их из системы.

Для пользователей guest и system пароли отсутствуют. Это означает, что вход в систему под этими аккаунтами невозможен.
